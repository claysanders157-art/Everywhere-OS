1.0.1 (2026/04/20)

Fixes
(common)
00001 Font rendering shows identical blocks for all characters - replaced with proper 8x8 bitmap font (ASCII 32-126)
00002 Snake game has no controls - added arrow key input (up/down/left/right scancodes)
00003 Notes window cannot receive keyboard input - added window focus system with click-to-focus
00004 Taskbar buttons for minimized windows are not clickable - added HandleTaskbarClick to restore windows
00005 All keyboard input always goes to Shell regardless of context - input now routed by active window
00006 Functions lack documentation - added DDK-style /*++ --*/ comment blocks to all routines
00007 Kernel reboots infinitely on startup - SetMode13h used int 0x10 BIOS call in protected mode causing triple fault
00008 GetKeyChar consumes mouse bytes from PS/2 buffer, misinterpreting them as scancodes (e.g. ESC) - added status bit 5 check
00009 UpdateMouse consumes keyboard bytes from PS/2 buffer - added status bit 5 check to only read mouse data
00010 InitMouse leaves stale ACK bytes in PS/2 buffer - added flush loop after initialization
00011 Multiboot header does not request video mode - added 320x200x8 video mode fields so GRUB sets mode 13h before kernel entry
00012 Screen flickers every frame - added double buffering with off-screen back buffer and FlipBuffers copy to VGA
00013 Entire OS is a single kernel.c file - split into multi-module layout
(mouse)
00017 Mouse cursor is a shapeless red triangle that disappears on dark backgrounds - replaced with proper 12x8 arrow cursor with black outline and white fill
00020 Mouse cursor still too small and poorly shaped for 640x480 - enlarged to 16x16 with classic arrow shape, proper tail split, and full black outline
00021 Mouse cursor is laggy and not smooth - UpdateMouse now drains all pending PS/2 bytes per frame instead of one byte per call
00022 inb(0x64) read twice per call causing status register to change between reads - status now read once into a local variable
00023 Corrupt mouse packets cause cursor jumps - added bit 3 validation and overflow flag discard
00024 Mouse cursor position drifts from host cursor in VM - configured PS/2 mouse with 1:1 scaling (0xE6), max resolution 8 counts/mm (0xE8 0x03), and 200 samples/sec (0xF3 200)
00025 QEMU opens in a tiny window instead of fullscreen - added -full-screen flag to QEMU launch command in Makefile


Fixes
(window)
00026 Window dragging is jerky and lags behind mouse - disabled physics updates during drag so window follows cursor directly

(mm)
00027 No memory manager - kernel has no way to dynamically allocate memory - implemented non-paged pool in base/ntos/mm: MmInit parses the Multiboot v1 memory map to find the largest usable physical region above 2 MB, MmAllocatePool performs first-fit allocation with block splitting, MmFreePool returns blocks with immediate bidirectional coalescing of adjacent free blocks
00028 Pool corruption would be silent - every block header carries a Magic sentinel (MM_POOL_TAG_ALLOC / MM_POOL_TAG_FREE); MmFreePool spins on magic mismatch to surface double-free and bad-pointer bugs immediately


Features
(window)
00014 Added fullscreen button to window title bar - click to expand window to fill screen, click again to restore original size
00015 Dragging is disabled while a window is fullscreen to prevent accidental moves
00016 Window physics (velocity/bounce) are skipped while fullscreen

Features
(video)
00018 OS runs at tiny 320x200 resolution - upgraded to 640x480x8 via VESA, multiboot header now requests 640x480, framebuffer address read from GRUB multiboot info
(kernel)
00019 Keyboard input routing logic lives in kernel.c - moved to shell/explorer/input.c where it belongs as userspace input handling
(mouse)
00026 Mouse cursor is slow and laggy because it's polled once per frame - converted to interrupt-driven via IRQ12 ISR, mouse data now processed immediately on arrival
(hal)
00027 No interrupt infrastructure exists - added base/hals/halx86/ with IDT setup, 8259 PIC remapping, and IRQ12 assembly stub matching WS2003 HAL layout
00028 Interrupt code was accidentally and incorrectly (according to me) placed in ke/ - moved to base/hals/halx86/ to match proper architecture (HAL separate from kernel executive)
00029 OS reboots on startup after HAL init - no GDT defined so IDT used unknown selector 0x08, added explicit GDT in entry.asm with flat code/data segments
00030 Unhandled IRQs (timer, etc.) triple-fault because IDT entries were null - added IrqIgnoreStub default handler for all 256 vectors
00031 PIC mask restore after remap could leave stray IRQs unmasked - now masks ALL IRQs first, then only unmasks IRQ2 (cascade) and IRQ12 (mouse)
(fs)
00032 No disk filesystem existed - added EVRYFS under base/fs/evryfs/ (ATA PIO driver in ata.c, filesystem logic in evryfs.c); on-disk layout: LBA 0 superblock, LBA 1 root directory (12 entries), LBA 2+ sequential data; auto-formats a fresh disk on first boot
(shell)
00033 No way to create files from the shell - added mktest command that writes "Hello, World!\n" to hello.txt on the EVRYFS volume and shows "Created hello.txt" in the shell output line
00034 No way to open the file browser from the shell - added files command that opens the Files window

Features
(tests)
00035 No regression test suite exists -- pool bugs from future commits would go undetected; added base/ntos/mm/tests/ containing a standalone bare-metal test kernel (entry.asm + mmtest.c); 20 test cases across five groups (allocation fundamentals, pool statistics, coalescing, structural invariants, exhaustion/recovery) exercise MmAllocatePool and MmFreePool with MM_POOL_TAG sentinels and report PASS/FAIL over COM1 serial port; runs under QEMU -nographic via 'make test'
(explorer)
00035 No file browser window existed - added Files window (shell/explorer/files.c) showing EVRYFS directory listing with filename and size per row; accessible via files shell command or taskbar
